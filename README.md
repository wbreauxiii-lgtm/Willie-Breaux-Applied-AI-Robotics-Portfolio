## Learning Journey

My AI and robotics coursework has moved from foundations into more applied systems. Earlier work focused on Python, data analysis, machine learning, and computer vision notebooks. Current work expands into deep learning, NLP product design, AI applications, Raspberry Pi and robotics platforms, and AI-assisted system planning.

Across these projects, I practiced turning technical work into documented, reviewable project folders. I also learned that a strong AI portfolio is not just code. It needs problem statements, methodology, results, limitations, data notes, and clear instructions for someone else to review the work.

## Public Repository Notes

This repository is meant to be public and employer-ready. Private files, live credentials, virtual environments, dependency folders, trade-secret design documents, and large generated artifacts were intentionally excluded. Where a project depends on a file that should not be published, the project README includes a TODO or instruction instead of inventing a result.
