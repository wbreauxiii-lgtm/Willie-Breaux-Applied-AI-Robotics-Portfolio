# Robotics, ROS, and Raspberry Pi Portfolio

## Overview

This folder contains current-semester robotics platform portfolio work. The main markdown file documents Raspberry Pi, Sense HAT, PiCar-X, ROS/platform concepts, and related AI/robotics learning activities.

## Problem Statement

Robotics projects require connecting software, hardware, sensors, and documentation. This coursework explores how embedded computing platforms such as Raspberry Pi can support robotics learning and applied AI systems.

## Tools and Technologies

- Raspberry Pi
- Python
- Thonny IDE
- Sense HAT
- PiCar-X
- ROS/platform concepts
- Presentation and report documentation

## Approach / Methodology

The project documents module-by-module robotics activities, including setup, hardware interaction, lab work, troubleshooting, and reflection. The included `build_pptx.py` script is an existing source file related to presentation generation.

## How to Run

1. Review `Willie_Breaux_Portfolio_ITAI2374.md` for the full coursework portfolio.
2. Review `build_pptx.py` as the included script file.
3. Use the PDF/PPTX presentation files as supporting school-submission artifacts.


## Results / Output

The portfolio documents hands-on results such as Raspberry Pi setup, Sense HAT interaction, camera/audio work, PiCar-X activities, and robotics platform learning.

## What I Learned

- How hardware and software connect in robotics labs
- How Raspberry Pi can support sensors, camera work, and robot platforms
- How ROS/platform concepts relate to applied AI and robotics systems
- How to document team and individual lab contributions

## Notes / Limitations

Some robotics work depends on physical hardware and classroom lab setup. The repository includes documentation and source artifacts available in the workspace, not a full hardware replication package.
