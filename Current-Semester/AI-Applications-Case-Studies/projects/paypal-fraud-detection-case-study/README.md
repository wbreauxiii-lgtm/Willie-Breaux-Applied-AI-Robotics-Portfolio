# PayPal Fraud Detection Case Study

## What This Is

This case study looks at fraud detection where speed matters, false positives cost money, and attackers adapt fast. The core problem is not whether AI sounds impressive. The problem is whether a system can separate suspicious behavior from legitimate behavior at transaction scale without creating operational drag or eroding customer trust.

## What I Took From It

- fraud detection is a systems problem, not just a model problem
- signal quality matters as much as algorithm choice
- real-time scoring only helps if the downstream decision logic is disciplined
- human review still matters when the cost of a bad call is high

## Concepts In Play

- machine learning
- anomaly detection
- behavioral analytics
- feature engineering
- real-time scoring
- risk-based decisioning

## Why I Included It

This is one of the clearest examples in the portfolio of AI being useful because it solves an expensive operational problem under pressure. It also shows the tradeoff side: detection quality, customer friction, model drift, and the need to keep improving against adversaries.
